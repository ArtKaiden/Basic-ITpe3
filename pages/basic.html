3. Je website Hosten met Apache

Hiervoor gaan we Apache installeren

Wat is Apache?

Apache is een open-source en gratis webserver software die een cruciale rol speelt in de online beschikbaarheid van websites. Het kan worden gebruikt om websites te hosten.

Apache installeren

Voor het installeren gaan we eerst de lokale packages updaten om er zeker van te zijn dat de recentste apache2 gevonden wordt bij het commando dat hier na volgt

sudo apt update

Om apache te installeren moet je volgende commando uitvoeren in de terminal:

sudo apt install apache2 (Indien Y/N wordt gevraagd na het uitvoeren, voer dan Y in)

Apache werd nu succescol geinstalleerd. Dit programma host websites die wij maken. Standaard komt een website meegeleverd, op locatie var/www/html
Je kan dit uittesten door nu te navigeren naar localhost

Apache configureren

We kunnen onze website laten staan op de locatie waar we hem bouwden.(Dan moet de locatie toegevoegd worden aan de whitelist in het bestand etc/apache2/apache2.conf) Echter om netter te werken is het misschien aan de orde om onze gehoste websites te plaatsen op de standaard locatie van Apache. Namelijk onder het pad: ‚Äú/var/www/‚Äù. Je bent vrij in deze keuze.
Wij gaan hem verplaatsen.

  • Dit kunnen we gewoon doen met de copy and paste methode via de file browser
    Of via de terminal
    • kopieren (originele folder houden)

      sudo cp <website folder> <doel pad>

      vb: sudo cp ./Website ../../var/www

    • knippen (originele folder verplaatsen)

      sudo rm <website folder> <doel pad>

      vb=> sudo rm ./Website ../../var/www

Vervolgens moeten we een configuratie bestand aanmaken. Dit bestand is nodig zodat Apache weet welke bestanden moeten gehost worden en welke servernaam we er gaan aan toekennen.
Apache configuratie bestanden bevinden zich in:

etc/apache2/sites-available/
Wij gaan hier dus een nieuw configuratiebestand aan toevoegen.

  • Navigeer naar bovenvermelde folder

    cd etc/apache2/sites-available/

  • We gaan het default configuratie bestand kopieren en hergebruiken mits wat aanpassingen, dit spaart ons wat typwerk.

    Vul in plaats van <jouw bestand> een eigen naam voor het bestand in.

    sudo cp 000-default.conf <jouw bestand>.conf

    vb=> sudo cp 000-default.conf PE3.conf

  • Pas het nieuwe bestand aan met nano (het meegeleverde tekstverwerkingsprogramma van linux)

    sudo nano <jouw bestand>.conf

    vb =>sudo nano PE3.conf

    1. Naast ServerAdmin vul je JOUW emailadres in indien je wil dat je gecontacteerd kan worden wanneer iets fout gaat
    1. Naast DocumentRoot vul je de locatie in van jouw website. We hebben onze website verplaatsd naar var/www/Website/ dus dit is wat we hier gaan invullen. Als jouw website ergens anders staat moet je hier dat pad invullen. (vergeet zeker de / niet achteraan)
    1. Hieronder plaatsen we een nieuwe lijn met: ‚ÄùServerName‚Äù gevolgd door de URL die we willen gebruiken om onze website te bereiken.

      vb=> ServerName local.voorbeeld.com

    druk vervolgens op CTRL+S om het bestand op te slaan en CTRL+X om nano te sluiten

  • Activeer jouw gemaakte configuratie bestand in apache

    sudo a2ensite <jouw bestand>.conf

    vb=> sudo a2ensite WebsitePE3.conf

  • Deactiveer het standaard configuratie bestand in apache

    sudo a2dissite 000-default.conf

  • Als laatste: Herstart apache

    systemctl restart apache2

Als alles goed is zou jouw website nu toegankelijk moeten zijn via het typen van localhost als url in de browser.

In de volgende stap gaan we er voor zorgen dat elke pc op jouw lokaal netwerk aan deze website kan. Momenteel moet elke lokale PC het IP adres invullen van jouw ubuntu VM om de website te bezichtigen. We zouden dit graag makkelijker toegankelijk maken zonder dat een IP adres moet onthouden worden.

4. Je website lokaal toegankelijk maken
4a. Je website toegankelijk maken via de Hosts file

We kunnen op elke individuele pc die toegang moet kunnen krijgen tot deze website de hosts file aanpassen.

OPGEPAST. hier gaan we naar een folder die essentieel is voor de werking van jouw computer. Verwijder of wijzig niet zomaar bestanden.

Wat is het hosts bestand?

Een¬†hosts file¬†is een tekstbestand en bevat hostnamen en het IP-adres waarnaar die verwijzen. Bij een opzoeking van een domeinnaam wordt altijd eerst gekeken naar het hostsbestand op je computer, vooraleer nameservers bevraagd worden.¬†Het grote nadeel van hosts files is dat het statische bestanden zijn. Wijzigingen in bijvoorbeeld een IP-adres worden niet automatisch overgenomen.
source:
https://faq.nomeo.be/nl/kba155/nameservers-en-dns-records/nameservers/wat-is-een-hosts-file-en-wat-is-het-nut-ervan

Het hosts bestand aanpassen (Windows)

navigeer naar volgende locatie:

C:\Windows\System32\drivers\etc

hier vind je het bestand genaamd hosts (dit bestand heeft geen bestandstype)
open het bestand in notepad of een ander tekstverwerkingsprogramma en onderaan kun je een nieuwe lijn bijvoegen.


Eerst vul je het ip-adres in van het systeem waar je je website en apache op geinstalleerd hebt, gevolgd door de domeinnaam die je wilt gebruiken om deze website te bereiken.

Wanneer je nu navigeert naar jouw ingevulde domeinnaam ga je automatisch terecht komen bij het correcte ip-adres van jouw virtuele machine en dus bij jouw website.

Het hosts bestand aanpassen (Ubuntu)

In ubuntu bevind de hosts file zich op /etc/hosts

hier vind je het bestand genaamd hosts (dit bestand heeft geen bestandstype)
open het bestand via volgend commando

sudo nano /etc/hosts


Eerst vul je het ip-adres in van het systeem waar je je website en apache op geinstalleerd hebt, gevolgd door de domeinnaam die je wilt gebruiken om deze website te bereiken.

(Indien je de hosts file aanpast van de computer zelf met apache kun je het lokaal IP 127.0.0.1 invullen als IPnummer hier, in plaats van het publiek IP)

Wanneer je nu navigeert naar jouw ingevulde domeinnaam ga je automatisch terecht komen bij het correcte ip-adres van jouw virtuele machine en dus bij jouw website.

4b. Je website toegankelijk maken met een DNS server

Hiervoor gaan we een DNS server opmaken, we gaan hier voor een applicatie genaamd BIND gebruiken.

Wat is BIND?

Dit is een open-source programma die het mogelijk maakt om onze eigen DNS (Domain Name System) server te maken. Dit gaat dus een leesbare websitenaam omzetten naar het gerelateerde ip-adres.

Bind installeren

Voor het installeren gaan we eerst de lokale packages updaten om er zeker van te zijn dat de recentste Bind gevonden wordt bij het commando dat hier na volgt

sudo apt update

Om Bind te installeren moet je volgende commando uitvoeren in de terminal:

sudo apt install bind9 (Indien Y/N wordt gevraagd na het uitvoeren, voer dan Y in)

Bind configureren
configuratie bestand aanpassen

Net zoals we bij Apache het configuratiebestand moesten aanpassen, moeten we dit ook doen bij BIND.
Het bestand bevind zich op
/etc/bind/named.conf.local

Open een nieuw terminal venster en voer volgend commando uit

sudo nano /etc/bind/named.conf.local

in dit bestand schrijven we het volgende (vervang <jouw serverNaam> met het domein dat jij wilt gebruiken voor jouw website)

zone "<jouw serverNaam>" {
type master;
file "/etc/bind/zones/<jouw serverNaam>";
};

Run na opslaan nog even volgend commando om te controleren of er geen fouten in jouw bestand staan

named-checkconf /etc/bind/named.conf.options

Zones bestand aanmaken

In de zopas toegevoegde tekst verwezen we naar een zogenaamd ‚Äúzones‚Äù bestand, echter bestaat dit bestand nog niet.

Voor we dit doen hebben we eerst ons IP adres nodig.
Om dit te vinden voer je het volgende commando ui

ip a

en dan vind je jouw ip adres naast het woord inet, noteer deze ergens voor straks.

Nu zijn we klaar om het zones bestand aan te maken met volgende commandos:

sudo mkdir /etc/bind/zones

sudo nano /etc/bind/zones/<jouw serverNaam>

Schrijf nu het volgende in dit bestand, vervang zoals altijd zeker <jouw serverNaam>, <jouw email> en<jouw ip> naar jouw specifieke situatie.
plaats hier steeds een . achter jouw serverNaam, en vervang de @ in jouw email naar een .

$TTL 604800
@       IN      SOA     <jouw servernaam>.       <jouw email>( 
                                1 
                                604800 
                                86400 
                                2419200 
                                604800 )

@       IN      NS      <jouw servernaam>.
@       IN      A       <jouw IP>

Voer ook hier volgend commando uit om je code te controleren

named-checkzone <jouw serverNaam> /etc/bind/zones/<jouw serverNaam>

Bestand aanmaken om terug te vallen op externe DNS

voer volgend commando uit

sudo nano /etc/bind/named.conf.options

en voeg dit toe aan het bestand:
Deze lijnen kunnen reeds in commentaar aanwezig zijn. Je kan die lijnen ook gewoon uit commentaar halen door de schuine streepjes er voor te verwijderen.

options {
    forwarders {
        8.8.8.8;
        8.8.4.4;
    };
};
Herstart bind

systemctl restart bind9

DNS aanpassen (Windows)

Ga naar je settings via windows+i en navigeer naar Network and internet

Druk op Properties bovenaan net onder jouw huidige netwerkverbinding.

Druk hierna op de rubriek DNS server assignment

Pas dit aan naar manual en vul het ip-adres in van jouw virtuele machine waar je Bind op geinstalleerd hebt onder ipv4 bij Preferred DNS.

Druk hierna op save.

DNS aanpassen (Ubuntu)

De makkelijkste manier om dit aan te passen gebeurt via een nieuw programma die we moeten installeren. Dus we gaan eerst updaten met :

sudo apt update

gevolgd door het commando om resolvconf te installeren:

sudo apt install resolvconf

Daarna gaan we een bestand aanpassen via:

sudo nano /etc/resolvconf/resolv.conf.d/head

We voegen hier een lijn toe:

nameserver <IP van jouw systeem waar bind op staat>

en als laatste starten we resolvconf met:

sudo systemctl enable --now resolvconf.service

Nu is jouw server naam, die je steeds gebruikte bij de vorige stappen, toegankelijk voor elk apparaat van wie je de standaard DNS hebt aangepast naar de DNS van jouw apparaat waar Bind op staat.

4c. Je website lokaal toegankelijk maken zonder individuele configuratie met Avahi

Hiervoor gaan we een programma gebruiken genaamd avahi

wat is avahi?

Een open source programma die werkt met het mDNS (multicast DNS) protocol. Dit laat apparaten toe om een pagina te bereiken zonder dat daar een hele DNS server voor moet opgesteld worden.

Avahi installeren

Voor het installeren gaan we eerst de lokale packages updaten om er zeker van te zijn dat de recentste Avahi gevonden wordt bij het commando dat hier na volgt

sudo apt update

Om Avahi te installeren moet je volgende commando uitvoeren in de terminal:

sudo apt install avahi-daemon (Indien Y/N wordt gevraagd na het uitvoeren, voer dan Y in)

Nu kan iedereen op jouw lokale netwerk jouw pagina vinden via <jouw VMnaam>.local

Indien je liever een andere naam gebruikt om toegankelijk te zijn, volg dan de volgende stap.

Avahi configureren (optioneel)

We gaan een nieuw configuratiebestand aanmaken om mee te geven welke lokale domeinnaam we willen gebruiken. het bestand gaat zich bevinden op
/etc/avahi/services/

Start dus een nieuwe terminal en voer volgend commando uit:

sudo nano /etc/avahi/services/<jouw domeinNaam>.service

In dit bestand plaats je volgende code, vergeet zoals altijd niet om <jouw domeinNaam> te vervangen

<?xml version="1.0" standalone='no'?>
<!DOCTYPE service-group SYSTEM "avahi-service.dtd">
<service-group>
  <name><jouw domeinNaam></name>
  <service>
    <type>_http._tcp</type>
    <port>80</port>
  </service>
</service-group>
Herstart Avahi

systemctl restart avahi-daemon
